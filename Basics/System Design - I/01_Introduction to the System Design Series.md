# 🚀 Introduction to the System Design Series


## 🚧 The Core Problem with Learning System Design

The specific problem is that when you ask someone about System Design, they will usually just point you to some resources, saying, "read from here."

> "The problem with this approach is that there are certain terms that we simply do not know or understand yet."

For example, a resource might casually mention terms like **Load Balancer** or **DNS** in real-time, and we don't properly know what they mean.

**The messy cycle of learning:**

1. We encounter an unknown term in a System Design course.
2. We leave the course to separately read about **DNS**.
3. We go somewhere else to study what a **Load Balancer** is.
4. After reading from those external sources, we come back and resume the System Design course.

Because of this constant context-switching, the learning process becomes very messy.

Explaination all of these things in one place.
